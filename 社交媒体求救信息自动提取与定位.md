# 社交媒体求救信息自动提取与定位

## **一、统一任务要求**

本任务面向“AI+防灾减灾”课程实践，要求学生围绕真实或模拟防灾减灾场景，完成数据获取、数据处理、算法实现、方案设计、效果展示和课堂汇报等完整流程。每组需选择1个项目开展实践，最终形成可展示、可复现、可评价的项目成果。

## **二、统一提交成果**

**数据包：**包括原始数据、处理后数据、字段说明、数据来源说明和数据预处理过程说明。

**源码：**包括数据读取、数据清洗、模型训练、算法实现、结果可视化等代码；方案类项目可提交原型源码、流程图源文件或可视化实现文件。

**演示录像：**建议3—6分钟，展示项目背景、数据、代码或系统运行过程、核心结果和结论。

**技术方案：**建议2000—4000字，包含背景、问题定义、数据来源、技术路线、实施步骤、结果分析、应用价值和改进方向。

**汇报PPT：**建议8—12页，包含项目背景、问题分析、数据来源、技术路线、方法实现、结果展示、效果图、应用价值和总结。

## 三、项目：社交媒体求救信息自动提取与定位

**项目需求：**利用NLP方法从微博、短视频评论、新闻留言或模拟文本中提取灾害求救信息，并进行位置解析。

### **具体要求**

1. 构建不少于100条灾害求救文本样例。
2. 提取地点、人员、灾情、需求等关键信息。
3. 将文本信息结构化为表格。
4. 根据地点信息生成求救点分布图。
5. 分析该方法在应急救援中的应用价值。

### **提交成果**

- 数据包：灾害求救文本数据、标注样例和结构化结果表。
- 源码：文本清洗、关键词提取、命名实体识别或地理编码代码。
- 演示录像：展示文本输入、信息抽取和地图定位结果。
- 技术方案：社交媒体求救信息提取与定位方案。
- PPT：任务背景、NLP方法、定位结果和救援价值。
