# 数据字段说明

本文档说明项目中所有数据的**字段定义、数据来源、预处理流程**,
作为任务书要求的"数据包"中"字段说明、数据来源说明、数据预处理过程说明"。

---

## 一、数据来源说明

### 1.1 公开数据集调研结论

在动手构建数据前,我们调研了中文 NER 公开数据集生态:

| 数据集 | 规模 | 实体类型 | 领域 | 与本项目契合度 |
|--------|------|----------|------|----------------|
| **CLUENER2020** ([GitHub](https://github.com/CLUEbenchmark/CLUENER2020)) | 训练 10,748 / 验证 1,343 | 地址、姓名、机构、职位、游戏等 10 类 | 新闻 | LOC/PER 可复用,但无 DIS/NEED |
| **Weibo NER** ([ModelScope](https://www.modelscope.cn/datasets/damo/weibo_ner)) | 训练 1,350 / 验证 269 / 测试 270 | GPE/LOC/ORG/PER(名+NOM) | 微博 | 文风接近,但无 DIS/NEED |
| **MSRA NER** | 训练 4.6 万 | LOC/PER/ORG | 新闻 | 无 DIS/NEED |
| **新浪微博地震舆情数据库** ([期刊](http://zzfy-eq.cn/cn/article/doi/10.11899/zzfy20230421?viewType=HTML)) | — | 情感/主题 | 微博地震 | 面向情感分析,非实体抽取 |
| **CrisisLex / TLLMR4CM** | — | LOC 等 | 英文灾害推特 | 跨语言迁移代价大 |

**调研结论**:
- ✅ **通用的中文 NER 数据集是存在的**(CLUENER、Weibo NER 等),可直接下载,涵盖 LOC/PER 等通用实体;
- ❌ **但专门标注"灾情(DIS)+ 需求(NEED)"的灾害求救 NER 公开数据集,目前确实没有现成的**——多数灾害舆情数据做的是情感分析、主题挖掘、灾损估计,而非我们这种面向救援调度的实体抽取;
- ⚠️ 现有数据集即使能复用 LOC/PER,也无法覆盖本项目任务特有的 DIS/NEED 两类核心实体。

### 1.2 本项目的数据策略

基于上述调研,本项目采用**"人工种子样本 + 模板化数据增强"**的半合成策略,原因:

1. **覆盖任务专属实体**:公开数据集缺少 DIS/NEED,半合成数据能精确定义并覆盖 4 类实体(LOC/PER/DIS/NEED);
2. **标注 100% 准确**:由于数据由模板渲染生成,结构化标注天然正确,可经程序自动转换为 BIO 标签,**避免了人工逐字标注的标错风险**;
3. **可控性强**:能精确控制灾种分布(地震/洪水/台风/泥石流/火灾)、实体覆盖、紧急程度比例;
4. **可复现**:固定随机种子,数据生成过程完全可重现;
5. **隐私合规**:无真实姓名/电话/住址,适合课程项目展示。

**诚实说明的局限**(已在技术方案"改进方向"中写明):
- 模板生成句式略偏标准化,真实微博更口语化、含错别字;
- 测试集与训练集同分布,模型 F1 可能略高于真实场景;
- 改进路径:接入真实微博 API + 半监督学习,或先用 CLUENER/Weibo NER 预训练再微调(可作为后续工作)。

### 1.3 数据规模

| 来源 | 数量 | 构建方式 | 风格 |
|------|------|----------|------|
| 手工种子样本 | 68 条 | 人工撰写 | 贴近真实微博/评论,覆盖地震/洪水/台风/泥石流/火灾 5 大灾种 |
| 模板扩增样本 | 220 条 | 灾种×地名×实体词模板渲染 | 句式多样、紧急程度均衡 |
| **合计** | **288 条** | | 满足任务书"≥100 条"要求 |

### 灾种分布

- **地震**:汶川、北川、雅安、九寨沟、玉树、积石山、泸定、花莲等典型震区
- **洪水/内涝**:郑州、新乡、鹤壁、信阳、涿州、邢台、余姚等(参考 2021 河南暴雨、2023 涿州洪水)
- **台风**:厦门、湛江、文昌、北海、宁德等沿海
- **泥石流/山体灾害**:舟曲、茂县、水城、盐源等山区
- **火灾**:重庆、西昌、长沙、青岛等

### 地名库数据来源

`china_geo_dict.json` 的经纬度来源于:
- 国家基础地理信息中心公开数据
- 高德地图坐标系(GCJ-02,火星坐标)
- **注**:经纬度为近似值(精度约 100m-1km),用于演示可视化,非工程级精度。

## 二、原始数据字段 (`data/raw/distress_messages_raw.jsonl`)

每行一个 JSON 对象:

```json
{
  "id": "seed_0001",
  "text": "救命啊!汶川映秀镇中心学校3楼塌了,我们一家三口困在里面,缺水和食物,电话13900001234",
  "entities": [
    {"type": "LOC",  "text": "汶川映秀镇中心学校3楼"},
    {"type": "DIS",  "text": "塌了"},
    {"type": "PER",  "text": "一家三口"},
    {"type": "NEED", "text": "水和食物"},
    {"type": "NEED", "text": "电话13900001234"}
  ],
  "urgency": "emergency",
  "source": "handwritten"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | str | 唯一标识 (`seed_XXXX` 或 `tmpl_XXXX`) |
| `text` | str | 求救文本(已清洗:去 URL、@、emoji、繁简归一) |
| `entities` | list | 结构化实体列表,每项 `{type, text}` |
| `entities[].type` | str | 实体类型: `LOC`/`PER`/`DIS`/`NEED` |
| `entities[].text` | str | 实体文本片段 |
| `urgency` | str | 紧急程度: `emergency`(紧急) / `high`(高) / `medium`(中) |
| `source` | str | 来源: `handwritten`(人工) / `template`(模板) |

## 三、标注数据字段 (`data/processed/{train,dev,test}.jsonl`)

由 `src/data/annotate.py` 从原始数据自动生成 BIO 逐字标签:

```json
{
  "id": "seed_0001",
  "text": "救命啊!汶川映秀镇中心学校3楼塌了...",
  "tokens": ["救","命","啊","!","汶","川",...],
  "tags":    ["O","O","O","O","B-LOC","I-LOC",...],
  "entities": [
    {"type": "LOC", "start": 4, "end": 16, "text": "汶川映秀镇中心学校3楼"},
    ...
  ],
  "urgency": "emergency",
  "source": "handwritten"
}
```

### BIO 标签体系

| 标签 | 含义 |
|------|------|
| `O` | 非实体(Outside) |
| `B-LOC` | 地点实体起始(Begin) |
| `I-LOC` | 地点实体内部(Inside) |
| `B-PER` | 人员实体起始 |
| `I-PER` | 人员实体内部 |
| `B-DIS` | 灾情实体起始 |
| `I-DIS` | 灾情实体内部 |
| `B-NEED` | 需求实体起始 |
| `I-NEED` | 需求实体内部 |

**共 9 个标签**(1 个 O + 4 类实体 × 2 前缀)。

### 数据划分

| 集合 | 数量 | 占比 | 用途 |
|------|------|------|------|
| `train.jsonl` | 201 条 | 70% | 训练模型 |
| `dev.jsonl` | 28 条 | 10% | 验证/调参/早停 |
| `test.jsonl` | 59 条 | 20% | 最终评估 |

划分采用固定随机种子(SEED=42)保证可复现。

## 五、地名库字段 (`data/geo/china_geo_dict.json`)

```json
{
  "汶川映秀镇": {
    "lng": 103.485,
    "lat": 31.006,
    "province": "四川",
    "level": "镇"
  },
  ...
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `lng` | float | 经度(GCJ-02) |
| `lat` | float | 纬度(GCJ-02) |
| `province` | str | 所属省/直辖市 |
| `level` | str | 级别:省/市/县/区/镇/乡/村/点/河/湖/江/旗/山/街道/路 |

共 **212 条**地名,覆盖:
- 21 个省/直辖市
- 54 个地级市
- 43 个县/区
- 27 个镇
- 9 条河流、若干典型灾害点

匹配策略:按地名长度降序做最长匹配,优先匹配更具体的地名。

## 六、数据预处理流程

```
原始文本 (含 @、#话题#、emoji、繁体等)
    │
    ▼ src/data/preprocess.py (clean_text)
[1] 去 URL (http/https/www)
[2] 去 @用户名
[3] #话题# → 话题词 (保留关键词)
[4] 去 emoji / 控制字符
[5] 繁简转换 (opencc t2s)
[6] 压缩重复标点 ("！！！" → "！！")
[7] 空白归一化
    │
    ▼ 清洗后文本 + 结构化实体
    │
    ▼ src/data/annotate.py (annotate_sample)
[1] 对每个实体文本,在清洗后文本中查找位置
[2] 长实体优先占用字符(避免重叠)
[3] 生成字符级 BIO 标签
[4] 输出 {tokens, tags, entities[{start,end}]}
    │
    ▼ 划分 train/dev/test (7:1:2, 固定种子)
```

### 关键设计

1. **清洗后再标注**:避免清洗破坏实体边界。原始文本和实体文本做相同清洗,
   保证一致性。

2. **长实体优先**:实体按长度降序处理,长实体(如"汶川映秀镇中心学校")
   优先占用字符,避免被短实体(如"汶川")截断。

3. **字符级标注**:中文 NER 在 BERT 中按字符切分(tokenize 时 `is_split_into_words=True`),
   BIO 标签也是字符级,与 BERT 的字 token 一一对应。

4. **可复现性**:所有随机操作(数据增强、划分)都使用固定种子 SEED=42。

## 七、数据统计

### 实体类型分布(全量 288 条)

| 类型 | 数量 | 平均每条 | 占比 |
|------|------|----------|------|
| LOC (地点) | 588 | 2.04 | 33.2% |
| PER (人员) | 290 | 1.01 | 16.4% |
| DIS (灾情) | 329 | 1.14 | 18.6% |
| NEED (需求) | 601 | 2.09 | 33.9% |
| **合计** | **1808** | 6.28 | 100% |

### 紧急程度分布

| 程度 | 数量 | 占比 |
|------|------|------|
| emergency (紧急) | 127 | 44.1% |
| high (高) | 115 | 39.9% |
| medium (中) | 46 | 16.0% |

紧急程度较高的样本占多数,符合求救信息的真实分布(求救者多处于紧急状态)。
