## 第 6 页 · NLP 核心方法:BERT NER

**标题**:BERT + CRF 序列标注

**模型结构图**:
```
输入字符 → [BERT 编码器] → [Linear 768→9] → [CRF] → BIO 标签
              ↑
        bert-base-chinese
        (1.02 亿参数)
```

**为什么用 BERT?**(三点):
- ✅ 中文预训练,理解中文语义
- ✅ Transformer 自注意力,捕获长距离上下文
- ✅ 少样本友好(288 条即可训练)

**为什么加 CRF?**(一点):
- ✅ 约束标签转移(避免"O → I-LOC"等非法情况)

**讲解要点**:BERT 做语义理解,CRF 做标签约束,二者互补。

## 第 8 页 · 结果展示:规则 vs BERT

**标题**:深度学习显著优于规则方法(F1 提升 64 个百分点)

**对比柱状图**(F1):
```
                F1
规则基线   ████████             0.3087
BERT+CRF   █████████████████████ 0.9511  ← 实测 (测试集 59 条)
```

**分类型对比表**(实测精确值):

| 类型 | 规则 F1 | BERT F1 | 提升 |
|------|---------|---------|------|
| LOC | 0.6117 | 0.9532 | +34 |
| PER | 0.2901 | 0.9402 | +65 |
| DIS | 0.2985 | 0.9231 | +62 |
| NEED | 0.1214 | 0.9710 | **+85** |
| **总体** | **0.3087** | **0.9511** | **+64** |

**讲解要点**:BERT 在所有类型上都显著优于规则,尤其 NEED 提升最大(+85 个百分点)——这正是深度学习的核心价值(规则无法穷举所有需求表述)。

**配图建议**:`outputs/reports/training_curve.png`(训练曲线)+ `metrics.json` 数据绘制对比图
